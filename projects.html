Active Directory Projects

Active Directory Lab

I put together an Active Directory home lab with a Windows 2019 Server and a Windows 10 Enterprise workstation. On this lab I configured DHCP and DNS so machines that I connected to the domain could receive an ip and have internet access. I also created users and set a password but I had set it up so then upon logon the user would change the password to something they could remember. I also added groups like IT, Security, and Workstations. I had pictured an imaginary scenario of this being in a building with multiple floors and so from that scenario in mind that was how I structured my organization units, creating floor 1 floor 2 and so and within them contained sub OUs of the Users and Computers. In this lab I also explored around with Group Policy Objects and created a GPO so all users on floor 1 had a WindowsXP wallpaper. In the Domain Controller I also had tested with shared resources and created a folder and configured permissions to allow it to be a shared resource strictly for the IT Group. Domain User Add/Checker

I created this PowerShell script to be able to add Users and essential information about the user in a faster way within the command line. This script was not only just for adding but it could be used as a checker to view information on a existing user, if you input an existing user you will get that users information and will be told the user exists. Whether a new user or existing user you input or retrieve the following information:

  • Enabled Status
  • Common Name
  • Given Name
  • SurName
  • User Principle Name
  • Department
  • Title
  • Email
  • Office Phone
  • Primary Group
  • Member Of (Groups)
  • When Created
  • When Changed
  • Password Not Required Status
  • Password Never Expires Status
  • Password Last Set Date
  • User Description

When adding a new user after adding the essential information in the password section you input it and you get prompted a box for confirmation that you are sure of that password and when it comes to groups you get a list of the current groups in the domain and you select the group you wish to add the user to. Retrieving Essential System Info via Remoting

I put together a script to get essential information from systems in the domain. With this script I had to start off by knowing how to locate current information relating to OS and the hardware from each system which I figured out by learning how to query WMI (Windows Management Instrumentation) using the Cim-Instance command to query the different classes. The WMI classes queried were:

  • Win32_LogicalDisk
  • Win32_OperatingSystem
  • Win32_Service
  • Win32_PhysicalMemory

Next was specifying the OU to retrieve the information from the systems in the OU, so I placed them into an array to be able to query each system using a loop. I then proceded to create a custom object using a hashtable with the properties I wanted to retrieve from each system which were:

  • System Name
  • IP Address
  • Operating System
  • Available Drive Space (GB)
  • Memory (GB)
  • User Profile Size (MB)
  • Stopped Services
