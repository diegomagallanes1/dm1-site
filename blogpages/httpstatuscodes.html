HTTP STATUS CODES

Http status codes are standard response codes provided by web servers to indicate the result of a client's request. The codes are grouped into five categories that each represent a different type of response. Here are the commonly encountered status codes.

Often Encounters:

200

Ok. Successful data retrieval

301/302

Redirects. Used for site migrations or temp links

404

Not found. URL is broken or doesn't exist

500

Internal server error. Server side code failing

These are the codes that are often encountered. The codes below give some more specific encounter types according to the categories. And for a more in specific in depth view of other possible status code encounters there is a link at the bottom!

The 100s:

Response codes from the 100s means that the request was simply recieved

100

client should continue the request

101

server agrees to switch protocols

102

request recieved by server but no status was available at the time of the response

103

preloading sources while server prepares a response

The 200s:

Response codes from the 200s confirms that the server successfully processed the request

200

Ok. Successful request, content returned depends on the request method. GET: resource retrieved. POST: data submitted

201

Created. A new resource was created. Typically the response sent after POST requests, or some PUT requests

204

Request successful but no content is returned

The 300s:

Response codes from the 300s often means redirection is involved

301

Moved Permanently. The URL of the requested source has been moved to a new URL

302

Found. The resource temporarily resides at another URL, but the original URL should still be used in the future

303

The server sent this reponse to direct the client to get the requested resource at another URI with a GET request

304

The requested resource hasn't changed since the last request, so the client can used the cached version

305

Use Proxy. Indicates that a requested response must be accessed by a proxy.

307

Temporary redirect. Similar to 302, but ensures the same http method is used for the redirect

308

Permanent redirect. This means that the resource is now permanently in another URI, speicifed by the location response header.

The 400s:

Reponse codes from the 400s indicate that the client made a mistake in the request.

400

Bad request. The server couldn't process the request due to invalid syntax or missing information

401

Unauthorized. Authentication is required, but the client hasn't provided credentials to get the requested response

403

Forbidden. The server understands the request, but refuses to authorize it

404

The server can't find the requested resource

405

Method not allowed. The request method is known by the server but not supported by the target resource

429

Too many requests. The client has sent too many requests in a given amount of time (rate limiting)

The 500s:

Reponse codes from the 500s indicate that the server failed to process a valid request

500

Internal server error. The server has encountered a situation it does not know how to handle

501

Not implemented. The server does not support the request method or functionality

502

Bad gateway. The server while working as a gateway or proxy recieved an invalid response from an upstream server

503

Service unavailable. The server is temporarily unavailable

504

Gateway timeout. The server acting as a gateway or proxy, didn't get a reponse in time from an upstream server
